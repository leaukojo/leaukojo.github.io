How to Flash ▾
ECU A — DFU (USB)
  1. Fetch or load the firmware above.
  2. If ECU A is running its app: click Trigger DFU via Serial, then Connect.
    If it is already in DFU mode: click Connect directly.
  3. Click Flash and Verify ECU A.
ECU B / C / D — CAN-FD bootloader
  1. Fetch or load the firmware above.
  2. ECU A must be running its application (not in DFU mode) to act as CAN gateway.
  3. Click Connect Serial and select ECU A's serial port.
  4. Tick the ECUs to program, then click Flash and Verify ECU B/C/D.

First-Time Setup / Recovery

This wizard flashes firmware to all four ECUs. Use this mode for a newly fabricated board, or to recover a board where ECU A is stuck in DFU mode.

ECU A must be in DFU mode before you begin. If it is not, use the Update RAMN mode instead.

Flashing CTF Firmware —

This is a CTF challenge firmware. A few things to know before you proceed:

  • While CTF firmware is running, reflashing from this tool is not possible. Use the reset tool when you're done.
  • This is fully reversible — restore standard firmware any time using the post-CTF board reset tool.

Logarithmic vs Linear ECU B

Default is Logarithmic — correct for most boards.

How to check if you have the wrong type installed:

  1. Run the RAMN vehicle simulation.
  2. Look at the ECU A display.
  3. Center the steering wheel.
  4. If STEER shows 0%: the firmware is correct.
  5. If STEER does not show 0%: re-flash ECU B with the other type.

Browser Support

This tool uses WebUSB (ECU A / DFU) and Web Serial (ECU B/C/D via CAN). These APIs are available in:

  • Chrome 89+ (desktop: Windows, macOS, Linux)
  • Edge 89+ (desktop)
  • Chrome for Android — requires a USB OTG adapter

Firefox and Safari do not support these APIs.

The page must be served over HTTPS or localhost.
